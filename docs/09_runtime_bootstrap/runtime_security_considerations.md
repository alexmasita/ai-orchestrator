Purpose

Bootstrap scripts execute with root privileges on newly created instances.

Security considerations must be carefully managed.

Execution Context

Bootstrap scripts run during instance initialization.

This means they:

run as root

have full system privileges

can modify system configuration

Security Risks

Improper bootstrap scripts could introduce:

arbitrary code execution

credential exposure

malicious infrastructure behavior

Security Controls

The system enforces several protections.

Deterministic Script Generation

Scripts are generated by the orchestrator.

User input cannot inject arbitrary shell commands.

Configuration Validation

Config values are validated before script generation.

Invalid configuration fails early.

Provider API Key Protection

The system never logs API keys.

Debug logs redact secrets.

No Dynamic Downloads

Bootstrap scripts avoid unverified external downloads.

Preferred pattern:

install deterministic dependencies

reference trusted sources

Network Exposure

Runtime services expose only:

Port	Service
8080	DeepSeek
9000	Whisper

Firewall rules should restrict access as needed.

Secret Handling

Secrets must never appear in bootstrap scripts.

Credentials must be handled via:

provider metadata

environment variables

secret managers

Recommended Hardening

Future improvements may include:

containerized runtime environments

signed bootstrap scripts

restricted service accounts

runtime network policies
