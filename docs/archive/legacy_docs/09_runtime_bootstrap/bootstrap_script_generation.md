Purpose

The bootstrap script is the mechanism used by ai-orchestrator to initialize a newly created GPU instance.

It is injected into the infrastructure provider at instance creation time and executed automatically when the machine starts.

The bootstrap script is responsible for:

preparing the runtime environment

installing required system dependencies

starting inference services

exposing deterministic service ports

enabling readiness checks

The script is generated deterministically by the orchestrator before instance creation.

Location in System Architecture
CLI
 ↓
run_orchestration()
 ↓
generate_bootstrap_script()
 ↓
provider.create_instance()
 ↓
bootstrap script executed on remote machine
Script Generation Entry Point

Bootstrap scripts are generated by:

ai_orchestrator.runtime.script.generate_bootstrap_script(config, models)

The orchestrator calls this function before creating the instance.

Example:

script = generate_bootstrap_script(config, models)
Deterministic Script Generation

Bootstrap scripts must be fully deterministic.

This guarantees:

identical infrastructure behavior across runs

reproducible deployments

deterministic tests

Determinism requirements:

Script content must be derived only from:

config

selected models

No timestamps

No random values

No environment-dependent output

Stable ordering of operations

Script Structure

The generated bootstrap script follows a fixed structure.

Example structure:

#!/usr/bin/env bash
set -e

# System preparation
apt update
apt install -y python3 python3-pip git

# Runtime setup
mkdir -p /opt/ai-orchestrator
cd /opt/ai-orchestrator

# Install runtime components
pip install llama-cpp-python
pip install faster-whisper

# Launch inference services
<deepseek startup>

<whisper startup>
Model-Specific Sections

Each model plugin contributes a section of the bootstrap script.

Plugin architecture allows:

plugins/
  deepseek_llamacpp.py
  whisper.py

Each plugin provides runtime requirements.

Example:

plugin.bootstrap_commands()

The script generator concatenates commands in a deterministic order.

Deterministic Ordering

Plugin execution order is defined by the plugin registry.

Example order:

deepseek_llamacpp
whisper

The registry guarantees deterministic iteration.

Script Delivery to Provider

After generation, the bootstrap script is passed to the provider:

provider.create_instance(
    offer_id,
    snapshot_version,
    instance_config={"bootstrap_script": script}
)

The provider injects the script into the instance creation request.

Provider Injection Mechanism

For the Vast provider, the script is injected via:

onstart

field in the instance creation payload.

Example payload:

{
  "onstart": "<bootstrap_script>"
}
Failure Handling

Bootstrap script generation may fail if:

plugin contract is violated

configuration values are invalid

These failures raise:

OrchestratorConfigError

before instance creation occurs.

Invariants

Bootstrap generation must satisfy the following invariants:

Script must be a string.

Script must be deterministic.

Script must not exceed the maximum size limit.

Script must expose fixed runtime ports.

Script must be valid bash.
